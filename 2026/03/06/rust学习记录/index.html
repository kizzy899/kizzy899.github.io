Rust Book 第一章：基础入门

🔧 核心概念

Cargo 是什么？

Cargo 是 Rust 的构建系统和包管理器，类似 npm/pip。

📝 常用命令

📁 项目结构

hello_cargo/
├── Cargo.toml # 项目配置文件
├── src/
│ └── main.rs # 源代码
└── target/ # 构建输出目录（自动生成）
└── debug/
└── hello_cargo (Windows: hello_cargo.exe)

注意：构建结果存放在 target/debug/ 目录，不与源代码混在一起。

💻 VS Code 连接云服务器

方案 1：VS Code Remote-SSH（推荐）🔥

直接在 VS Code 里连接云服务器，像本地一样开发：

\1. 安装扩展：VS Code → 扩展商店 → 搜 Remote - SSH → 安装

\2. 配置连接：

- Ctrl+Shift+P → Remote-SSH: Connect to Host...

- 输入 ssh user@your-server-ip

- 按提示输入密码/密钥

\3. 打开项目：连接后直接在服务器上打开文件夹

\4. 安装 Rust 扩展：在远程环境里装 rust-analyzer

优点：编译运行都在服务器上，环境一致，调试方便

🔣 变量声明

let apples = 5; // immutable（不可变）

let mut bananas = 5; // mutable（可变）

类型 关键字 说明
不可变变量 let x = 5; 默认不可变，绑定后不能修改
可变变量 let mut x = 5; 加 mut 后可修改

Rust 设计理念：默认不可变，安全优先。需要修改时显式声明 mut。

⌨️ 接收用户输入

let mut guess = String::new();

io::stdin()

​ .read_line(&mut guess)

​ .expect(“Failed to read line”);

🔍 详细解释：&mut guess

guess：

  • 是一个 String 类型的变量

  • 用来存储用户输入的文本

  • String::new() 创建一个空字符串

&mut：

  • & = 引用（Reference），类似指针但不做算术运算

  • mut = 可变（Mutable）

  • &mut guess = 可变引用，表示”借用 guess 的可变引用”

为什么用 &mut？

  • read_line() 需要修改 guess 的内容（把用户输入写进去）

  • Rust 的所有权规则：函数要修改传入的变量，必须传入可变引用

  • 如果只传 &guess（不可变引用），函数只能读不能改

  • 如果传 guess（所有权转移），guess 就不能再用了

类比理解：

guess → 一个笔记本

&guess → 让别人看你的笔记本（只能读）

&mut guess → 让别人在你的笔记本上写字（可以改）

guess → 把笔记本送给别人（所有权转移，你不能再用了）

⚠️ 处理潜在故障与结果

.expect() 作用：

  • read_line() 返回 Result<T, E> 类型（可能成功或失败）

  • .expect() 处理 Result：

  • 成功 → 返回内部值

  • 失败 → panic 崩溃，并显示传入的错误信息

为什么可能失败？

  • 用户输入无效 UTF-8 字符

  • 标准输入流出错

  • 系统资源问题

更优雅的处理方式（后续学习）：

match result {

​ Ok(_) => println!(“读取成功”),

​ Err(e) => println!(“读取失败：{}”, e),

}

第三章：变量类型与格式化

一、 标量类型 (Scalar Types)

标量类型代表一个单独的值。Rust 有四种主要的标量类型：整型、浮点型、布尔型和字符型。

1. 整数类型 (Integer Types)

位数 有符号 (i) 无符号 (u) 范围说明 (以 n 位为例)
8-bit i8 u8 $-2^{n-1}$ 到 $2^{n-1} - 1$
16-bit i16 u16 例如 u8: $0$ 到 $255$
32-bit i32 u32 默认类型，性能最优
64-bit i64 u64 常用语大数据量场景
128-bit i128 u128 极大数据量
arch isize usize 依赖运行机器架构 (64位或32位)

[!TIP]

选择建议： 如果不确定，通常直接使用 i32 即可。usize 主要用于数组索引或集合的大小计数。

2. 浮点类型 (Floating-Point Types)

Rust 的浮点数符合 IEEE-754 标准：

  • f32: 单精度浮点数。
  • f64: 双精度浮点数（默认类型，在现代 CPU 上速度与 f32 几乎相同，但精度更高）。

3. 布尔与字符类型

  • bool: 占用 1 个字节，值为 true 或 false。
  • char: 占用 4 个字节。
    • 注意： 不同于 C/C++ 的 1 字节，Rust 的 char 代表一个 Unicode 标量值。
    • 范围：支持拼音、中文、日文、韩文、Emoji 以及零宽间隔等。

二、 复合类型 (Compound Types)

复合类型可以将多个值组合成一个类型。

1. 元组 (Tuple)

  • 特点： 长度固定，元组内成员类型可以不同。
  • 访问方式： 解构（Destructuring）或点号（.）索引。

三、 核心避坑指南 ⚠️

1. 单引号 vs 双引号

在 Rust 中，引号的使用非常严格：

  • 单引号 'a': 用于 char 类型（单个字符）。
  • 双引号 "abc": 用于字符串字面量 &str。

2. 格式化输出 (Formatting)

Rust 使用宏（Macro）处理输出，通过 {} 占位符自动处理类型转换。

常用宏 用途
println! 输出到控制台并换行
print! 输出到控制台不换行
format! 返回格式化后的 String，不直接打印

四、 类型推断 (Type Inference)

虽然 Rust 是静态强类型语言，但编译器非常聪明：

  • 自动推导： 大多数情况下，编译器能根据赋值自动确定类型。
  • 显式标注： 当编译器无法推导（如 parse 字符串为数字）或为了代码可读性时，需要显式标注。

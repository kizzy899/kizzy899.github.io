Move 与 Aptos 开发笔记

Move 是面向数字资产的智能合约语言。它通过资源类型、所有权和能力约束减少资产被复制、重复使用或意外销毁的风险。Aptos Move 在基础 Move 语言之上提供账户、对象、事件和框架模块。

语言核心思想

  • 资源安全：具有 key 能力的资源可进入全局存储，不能被隐式复制或丢弃。
  • 所有权转移：值被移动后，原变量不能继续使用。
  • 显式能力：copy、drop、store 和 key 控制类型允许执行的操作。
  • 模块隔离：资源只能由定义它的模块直接创建和破坏。
  • 静态验证：大量错误在编译阶段被拒绝，便于形式化分析。

模块、入口函数与脚本

模块用于定义结构体、函数、常量和事件，是长期部署的代码单元。入口函数可由交易直接调用。脚本是一次性交易逻辑，可以调用公开模块函数，但新项目通常优先使用模块中的 entry 函数组织入口。

能力说明

能力 作用
copy 允许按值复制
drop 允许值在作用域结束时被丢弃
store 允许作为字段存入其他结构体
key 允许作为顶层资源进入全局存储

普通整数和布尔值通常可复制与丢弃。资产类结构体不应随意拥有 copy 或 drop，否则可能破坏唯一性约束。

值移动与借用

不可变引用使用 &T，可变引用使用 &mut T。同一数据在存在可变借用时不能再被其他引用访问，这一约束用于避免并发修改和悬空引用。

基础类型与字符串

Move 常用基础类型包括 bool、u8、u16、u32、u64、u128、u256、address 和 vector<T>。Move 不提供普通浮点数，金融计算通常采用整数定点数。

不同整数位宽之间不能隐式运算，需要显式转换并确认目标类型能够容纳原值。

依赖下载故障

首次编译可能需要拉取 Git 依赖。出现网络失败时，应依次检查代理、DNS、Git 是否可用、依赖仓库地址和分支是否存在。已有完整缓存时，可尝试跳过最新依赖获取：

该参数不能修复缺失或损坏的缓存。若需使用本地依赖，应在 Move.toml 中明确配置本地路径，并固定经过验证的版本。不要直接在依赖缓存目录中执行破坏性重置，因为缓存可能被多个项目共享。

单元测试

测试应覆盖正常路径、重复初始化、资源不存在、权限不足、整数边界和中止码。发布前还应执行格式化、静态检查和本地网络交互测试。

发布流程

发布前必须确认网络、发布账户、命名地址、Gas 预算和依赖版本。模块升级还需要检查兼容性规则，避免修改已发布资源的存储布局。

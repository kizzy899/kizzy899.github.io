项目笔记

整体

微服务+动静分离架构

后台需要启动7个微服务，其中gateway是网关，其他6个微服务通过它转发来访问。

Openresty直接提供静态页面的访问，同时将页面上动态数据有关的请求转发给微服务gateway，做到动静分离。

backend：Java后端，Springcloud微服务架构，提供前台需要的接口

account：账户服务，用户注册登录、个人中心信息维护

commons：一些公共jar包，被其他项目maven方式引入（无需启动）

gateway：网关，所有的前台页面请求都要经过网关路由到后端服务

modules：一些PO，VO和公共使用的对象，jar包，被其他项目maven方式引入（无需启动）

notice：消息服务，站内收发消息

order：订单服务，生成订单

payment：支付服务，维护支付信息

storage：存储服务，项目内的数据读写需要经过本微服务

stroke：行程服务，维护行程数据

web：前台h5界面，需要使用Openresty或Nginx来供浏览器访问

2.启动后台

默认情况下，所有配置文件已经修改指向虚拟机环境，如果使用的是自己的其他环境（云主机

等），注意修改所有项目下的yml文件里对应的ip和密码等

通过idea启动以下Springboot程序：account、gateway、notice、order、payment、storage、stroke

工具

工具方法列表及作用

1. HTTP 相关工具

  • getHeaderValues(Map<String, List> headerMap, String key)

  • 作用：从HTTP请求头Map中获取指定key的值

  • 用途：处理HTTP请求头信息

2. ID生成工具

  • getWorkerID()

  • 作用：生成唯一的工作ID

  • 用途：使用雪花算法生成分布式唯一ID

3. 加密工具

  • encodeMD5(String orginStr)

  • 作用：对字符串进行MD5加密

  • 用途：密码加密、数据校验等

  • fileSignature(byte[] file)

  • 作用：计算文件的MD5签名

  • 用途：文件完整性校验、去重等

4. 数字格式化工具

  • floatToStr(float ft)

  • 作用：将浮点数格式化为字符串（保留1位小数）

  • 用途：数字显示格式化

5. 对象转换工具

  • toVO(PO po)

  • 作用：将持久化对象(PO)转换为视图对象(VO)

  • 用途：数据层到表现层的数据转换

  • toVO(List poList)

  • 作用：批量将PO列表转换为VO列表

  • 用途：批量数据转换

  • toPO(VO vo)

  • 作用：将视图对象(VO)转换为持久化对象(PO)

  • 用途：表现层到数据层的数据转换

  • toPO(List voList)

  • 作用：批量将VO列表转换为PO列表

  • 用途：批量数据转换

6. 流处理工具

  • streamToString(InputStream inputStream)

  • 作用：将输入流转换为字符串

  • 用途：处理HTTP请求体、文件读取等
